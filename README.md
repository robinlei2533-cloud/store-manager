# 门店管家 — 使用指南

## 快速启动（3步搞定）

### 第1步：启动项目
```bash
cd /workspace
./start.sh
```
> 首次运行会自动安装依赖，之后直接启动。

### 第2步：打开浏览器
启动后终端会显示地址，通常是：
```
http://localhost:5173
```

### 第3步：登录
- **本地演示模式**下，输入**任意邮箱和密码**即可登录
- 系统会预置演示数据（8家门店、13条拜访记录、8个粉丝、6种物料、3场活动、5份评估等）
- 所有操作（增删改查）的数据保存在浏览器本地

## 六大功能模块

### 1. 数据看盘 `/dashboard`
- 8个统计卡片：门店数、拜访数、今日拜访、粉丝数、扫码次数、进行中活动、物料种类、低库存预警
- 近30天拜访趋势折线图
- 门店等级分布饼图（A/B/C级）
- 活动概览列表
- 物料库存预警列表（低库存自动标红）
- 最近拜访记录表
- 最近扫码积分记录

### 2. 门店管理 `/stores`
- 门店列表（搜索、按等级筛选）
- 新增/编辑门店（名称、地址、GPS经纬度、等级、连锁信息、联系人）
- 门店详情（基本信息 + 拜访历史记录）

### 3. 拜访管理 `/visits`
- 拜访列表（按日期、状态筛选）
- 新建拜访：选择门店 → 录入动销数据（产品+销量+销售额+库存）→ 上传照片（货架/展示/外观/产品）
- 拜访详情：查看动销数据表格 + 照片画廊

### 4. 店铺评估 `/evaluation`
- 6维度打分体系：
  | 维度 | 说明 |
  |------|------|
  | 销量/进货频次 | 门店近期销量和进货频率 |
  | 陈列质量 | 产品陈列规范度、排面丰富度 |
  | 地理位置与客流 | 门店位置、周边人流情况 |
  | 老板合作意愿 | 配合度、参与活动积极性 |
  | 连锁/扩张潜力 | 是否连锁品牌、扩张计划 |
  | 店铺外观装修 | 门头形象、店内整洁度 |
- 每项1-10分，总分60分
- 自动评级：平均≥8分=A级，≥6分=B级，否则C级
- 评估详情页含**雷达图**直观展示6维度得分
- 评估后自动同步更新门店等级

### 5. 活动管理 `/campaigns`
- 活动列表（卡片式展示，按状态分Tab：计划中/进行中/已结束）
- 创建活动：名称、类型（促销/新品推广/节日营销/品牌推广/门店拓展）、时间范围、预算、目标门店（多选）
- 活动详情3个Tab：
  - **活动概览**：基本信息 + 参与门店列表
  - **任务清单**：创建/编辑/删除任务，分配负责人，标记完成
  - **活动复盘**：填写复盘报告（总销售额、拜访数、扫码数、达成率、费效比、总结、改进建议）

### 6. 粉丝运营 `/fans`
- **粉丝列表**：等级（青铜→白银→黄金→铂金→钻石）、积分、贡献值，按积分排序
- **粉丝详情**：积分流水记录、等级信息
- **积分规则配置** `/fans/rules`：可配置各行为的积分值（完成拜访+10、上传照片+5、扫码积分+5等）
- **等级规则配置**：各等级最低积分和权益

### 7. 扫码积分 `/fans/scan`
- **码库管理**：为每个产品生成二维码（关联产品+门店+积分值），支持启停/删除
- **模拟扫码**：点击"模拟扫码"模拟消费者扫码，积分自动发放到门店粉丝账户
- **扫码记录**：查看所有扫码记录（时间、产品、门店、积分、粉丝等级）
- 扫码流程：消费者扫产品码 → 积分归入所属门店粉丝账户 → 粉丝积分增加 → 自动判断升级

### 8. 物料管理 `/materials`
- **物料目录**：CRUD物料（名称、SKU、分类、单位、单价）
- **库存看板**：当前库存 vs 安全库存，低于安全库存自动标红预警
- **入库管理**：记录入库（物料+数量+备注），自动增加库存
- **出库/领用**：申请领用（物料+数量+门店+原因）→ 管理员审批 → 审批通过自动扣减库存

## 从本地模式切换到云端模式

当需要多用户协作、数据持久化时，可切换到 Supabase 云端模式：

### 步骤
1. 创建 [Supabase](https://supabase.com) 项目
2. 在 Supabase SQL Editor 执行 `/database/migration.sql`
3. 在 Supabase Storage 创建 `visit-photos` 桶（公开访问）
4. 复制 `/workspace/frontend/.env.example` 为 `.env`，填入：
   ```
   VITE_SUPABASE_URL=https://your-project.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-key
   VITE_GOOGLE_MAPS_API_KEY=your-key
   ```
5. 重启项目即可自动切换为云端模式

> 配置 `.env` 后系统自动使用 Supabase，无需改代码。未配置时自动使用本地模式。

## 项目文件结构

```
/workspace/
├── start.sh                    # 一键启动脚本
├── frontend/                   # 前端项目
│   ├── src/
│   │   ├── components/         # 布局和通用组件
│   │   ├── pages/              # 页面（21个）
│   │   │   ├── login/          # 登录
│   │   │   ├── dashboard/      # 数据看盘
│   │   │   ├── stores/         # 门店管理
│   │   │   ├── visits/         # 拜访管理
│   │   │   ├── evaluation/     # 店铺评估 ← 新增
│   │   │   ├── campaigns/      # 活动管理 ← 新增
│   │   │   ├── fans/           # 粉丝运营 + 扫码积分 ← 增强
│   │   │   ├── materials/      # 物料管理
│   │   │   └── settings/       # 系统设置
│   │   ├── services/
│   │   │   ├── db/             # 本地数据引擎
│   │   │   ├── api.js          # 统一API（本地/云端自动切换）
│   │   │   └── supabase.js     # Supabase客户端
│   │   ├── stores/             # 状态管理
│   │   └── utils/              # 常量
│   └── .env.example            # 环境变量模板
└── database/
    └── migration.sql           # Supabase数据库迁移脚本
```
